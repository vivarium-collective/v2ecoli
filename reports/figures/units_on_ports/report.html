Summary

This PR (v2ecoli#105) puts the framework's coercing quantity[...] units onto model ports — so units are tracked through stores, computation and serialization — and adds a runtime mass-conservation check with a baseline behavior gate. Depends on the emitter change pbg-emitters#2 (serialize unit-bearing leaves on emit).

1 · Real units on the ports

  • The framework distinguishes float[unit] (metadata — a bare float at runtime) from quantity[...] (a real pint.Quantity). This PR moves the mass subsystem to real Quantities: the mass listener emits quantity[float,fg] (cell/dry/water/submasses/compartments), quantity[float,fL] (volume), quantity[float,fg] (growth), quantity[float,1/s] (growth rate); dimensionless ratios stay float.
  • Internal arithmetic stays in bare fg magnitudes and is wrapped only at the emit boundary, so the conversion is parity-preserving — see the before/after panel (curves coincide, final-step rel diff ~0).
  • ~10 process readers + every live-state consumer (reports, PDMP, workflow lineage/analysis, behavior tests) were migrated to a tolerant fg_magnitude() accessor (handles bare float or Quantity).

2 · Runtime mass-conservation check + verdict

New MassConservationListener (opt-in feature enable_features('mass_conservation')): each tick it balances the change in total cell mass against the net mass exchanged across the environment boundary by metabolism — the only legitimate mass source/sink.

Verdict: the baseline model conserves mass to ~1%. Over 120 ticks, Σ net boundary exchange ≈ Σ Δcell_mass to ~0.9% (see the convergence panel). The earlier apparent 55×/168× discrepancy was a check bug, not a model bug: environment.exchange is a cumulative store (metabolism adds to it each tick), so the listener now diffs it per tick; and the balance is against total cell mass (incl. water), not dry mass. Direct instrumentation confirmed metabolism's exchange is mass-balanced (the FBA's S·v=0 enforces it). The residual is a fixed ~1.2 fg cell-initialization offset that does not accumulate — the listener warms up past it.

3 · Behavior gate

  • test_baseline_conserves_mass — parametrized over seeds [0,1]; steady-state cumulative drift < 3% (healthy ~1%).
  • test_division_conserves_mass — division partitions the mother into two daughters with no exchange, so mother_cell_mass ≈ daughter1 + daughter2 (< 2%).
  • Fast suite: 317 passed, 0 failed. Fixed a stale kinetics unit-interop test (its contract moved to plain-float magnitudes). test_model_behavior (incl. daughters) green.
  • One nightly @slow test is red on a pre-existing chromosome-init assumption (initial state has 2 oriCs, test expects 1) — filed as #106, unrelated to units/conservation.
