Summary

This PR (v2ecoli#105) brings the framework's coercing quantity[...] units onto model ports — so units are tracked through stores, computation and serialization — and adds an opt-in runtime mass-conservation check. It depends on the emitter change pbg-emitters#2.

What changed

  • Tolerant accessors (quantity_helpers.py): magnitude_in / as_quantity / fg_magnitude — accept bare float[fg] or quantity[float,fg]; exact pass-through for bare numbers.
  • Mass subsystem → real Quantities: the mass listener emits quantity[float,fg] (cell/dry/water/submasses/compartments), quantity[float,fL] (volume), quantity[float,fg] (growth), quantity[float,1/s] (growth rate); dimensionless ratios stay float. Internal math unchanged (bare fg), wrapped only at the emit boundary; ~10 readers flipped to match the store type.
  • Live-state consumers migrated (reports, PDMP) so float(Quantity) no longer raises.
  • Emitter foundation (pbg-emitters#2): a Quantity on an emitted port is stripped to its magnitude under the same column name with the unit stored in Parquet file metadata — preserving downstream name-based queries.
  • MassConservationListener — opt-in (features=['mass_conservation'], OFF by default): per-tick Δdry_mass vs net metabolism exchange mass → emits a residual, warns (never raises). Not yet calibrated, hence opt-in.
  • Pilot wire: effective_elongation_rate (quantity[float,amino_acid/s]).
  • Docs: math-formulation opportunity map + units-on-ports sweep plan.

How to read the plots below

The mass conversion is parity-preserving by construction — the stored magnitudes are unchanged, only unit metadata is added. The before/after panels overlay the same 60-step baseline run on main (before) vs this branch (after); the curves should coincide, and the final-step relative difference should be ~0. That is the evidence the refactor changed representation, not behavior. At runtime cell_mass is now a Quantity(~1270, 'fg') instead of a bare float.
