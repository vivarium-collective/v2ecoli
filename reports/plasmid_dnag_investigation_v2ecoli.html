1. Overview & Verdict

The vEcoli plasmid investigation (vEcoli/reports/plasmid_replication_report.html) traced multigenerational DnaG depletion to four compounding regulatory-pipeline errors. v2ecoli is a port of the same parca code, so a priori all four are expected to be present. This report confirms each one against the actual parca-built sim data and source code in the v2ecoli plasmids branch, and notes the v2ecoli-specific differences that matter for the next round of work.

Verdict All four vEcoli root-cause findings reproduce in v2ecoli on plasmids. The regulatory-pipeline code and the raw flat-file inputs were ported verbatim, so the parca output has the same defects. Multigenerational DnaG depletion is therefore expected once the mechanistic_replisome gate is enforced over many generations — but we cannot yet observe that trajectory directly in v2ecoli (see §7).

2. Cause 1 — lexA → dnaG = −1.71 log₂

The NCA-derived fold change is present in the raw TSV and survives the parca fold-change pipeline intact. After TF-ID abbreviation resolution (lexA → PC00010) and gene-symbol resolution (dnaG → EG10239_RNA), it lands in sim_data.tf_to_fold_change with the same magnitude.

2.1 Raw TSV row

# v2ecoli/processes/parca/reconstruction/ecoli/flat/fold_changes_nca.tsv:2313
"lexA"	"dnaG"	-1.71

The TSV is loaded by knowledge_base_raw.py as part of the default attribute list (line 41: "fold_changes_nca.tsv"). Note that the manually curated fold_changes.tsv does not contain a lexA→dnaG row — the fold change is NCA-only, consistent with the vEcoli finding that it was never manually curated.

2.2 Confirmed in the built sim_data

# out/kb/simData_plasmid_patched.cPickle
sd.tf_to_fold_change["PC00010"]["EG10239_RNA"] = 0.30566   # 2^(−1.710)
len(sd.tf_to_fold_change["PC00010"]) = 53                 # PC00010 targets 53 RNAs total

53 targets include the whole SOS/DNA-damage regulon (EG10896_RNA/rpoD at −1.80, EG10747_RNA/lexA autoregulation, etc.). The SOS response itself is not modelled (no RecA cleavage of LexA in any v2ecoli process), so all 53 targets sit permanently near full repression at runtime.

Biological caveat (inherited from vEcoli analysis) The −1.71 log₂ is the maximum SOS de-repression signal observed in the EcoMAC expression matrix — dominated by norfloxacin experiments where LexA is fully cleaved. It does not represent constitutive lexA-dependent repression of dnaG during healthy growth. This is the same concern raised in the vEcoli report (§8, Step 1).

3. Cause 2 — LexA is 0CS (always-active)

v2ecoli's TF taxonomy has three categories: 0CS (no upstream signal — binding probability driven only by TF abundance), 1CS (single metabolite signal via equilibrium), 2CS (two-component phosphorelay). LexA is tagged 0CS in the condition flat-file and this classification propagates into sim_data unchanged.

3.1 Raw condition assignment

# v2ecoli/processes/parca/reconstruction/ecoli/flat/condition/tf_condition.tsv:17
"lexA"	"PC00010"	"minimal"	{}	"minimal"	{"EG10533_RNA": 0}	"0CS"

3.2 0CS binding law in the parca

The 0CS branch in promoter_fitting.py sets p_bound to min(1, TF_count / n_targets) · init_to_average: 

# v2ecoli/processes/parca/promoter_fitting.py:744–745
if tfType == "0CS":
    pPromoterBound[conditionKey][tf] = limited_tf_counts

At runtime, TF_count for PC00010 (the dimeric active LexA form) is bounded away from 0 by the equilibrium process, so LexA sits near-saturated on all 53 targets at every time step. There is no DNA-damage signal path to reduce it.

Sim_data consequence tf_to_tf_type["PC00010"] == "0CS" in the built sim_data. LexA is permanently ON. Every fold change listed in §2 is applied at full strength for the whole simulation, over every generation.

4. Cause 3 — Fold changes applied at cistron level

sim_data.tf_to_fold_change is keyed on cistron (i.e. RNA IDs of individual gene products), not on TU. The parca then expands each cistron to every TU containing it via transcription.cistron_id_to_rna_indexes(cistron). Each of those TUs receives the full fold change — regardless of whether the TF operator actually overlaps the TSS of that particular TU.

4.1 Cistron → TU expansion (regulation matrix)

# v2ecoli/processes/parca/promoter_fitting.py:695–712

    for cistron_id in sim_data.process.transcription.cistron_data["id"]

What this means for the rpsU-dnaG-rpoD operon The lexA fold change is keyed on the cistron EG10239_RNA (the dnaG cistron). In vEcoli, the parca expands this to all surviving TUs containing dnaG. In v2ecoli the dedup (see §5) leaves only TU00352 in sim_data, so the full −1.71 log₂ suppression lands on TU00352 — but v2ecoli's TU00352 is rpsUp1, a promoter whose TSS is 83 bp upstream of the LexA operator and therefore biologically constitutive. The FC is applied to a TU that LexA cannot physically regulate. Same bug class as vEcoli.

v2ecoli's starting DnaG count is 10 molecules/cell, slightly above the Schmidt 2015 proteomics value (6/cell) and in the same ball-park vEcoli reported (~6–10). 7. v2ecoli-specific gaps relative to the vEcoli investigation

7.1 No tf_tu_exemption variant

The vEcoli partial fix was a post-parca variant at ecoli/variants/tf_tu_exemption.py that absorbs a TF's delta_prob contribution into basal_prob for specified TUs, making them constitutive. v2ecoli currently has no variants mechanism at all:

  • grep -r tf_tu_exemption v2ecoli/ → 0 matches.
  • No v2ecoli/variants/ directory.
  • No variant-selection hook in save_cache / LoadSimData.
Implication There is no ready mechanism to apply even the vEcoli partial fix in v2ecoli. Either the variants pipeline must be ported, or the exemption must be implemented as an inline sim_data transform inside save_cache (quicker path, no variant registry). See §8.

7.2 Multigenerational depletion not yet directly observable

v2ecoli's single-cell pipeline completes generation 1 to division successfully and writes out/workflow/pre_division/pre_division_state.json.gz. However, attempts to run multigeneration simulations currently hit a bigraph-schema realize-time failure: 

ValueError: realize Array at path=('agents', '00', 'listeners', 'growth_limits'):
  np.array failed. encode type=list, len=39, first_row_type=ndarray,
  first_row_len=0, dtype=float64.

Cause (from the accompanying workflow investigation): listeners/growth_limits is a nested-dict store whose 39 sub-fields have mixed dtypes (int / float / bool arrays). At daughter _add, the schema resolver picks Array for the parent store, and np.array([...]) over the 39 empty per-field ndarrays fails with "inhomogeneous shape".

reports/multigeneration_report.py sidesteps this by rebuilding each daughter as a fresh Composite(doc, …) rather than structurally forking via _add. Running the multigen report with the plasmid cache (.venv/bin/python reports/multigeneration_report.py --cache-dir out/cache_plasmid) is therefore the first place the DnaG trajectory over generations can be observed in v2ecoli.

7.3 Parca ECOS solver (co-TU restoration) — untested in v2ecoli

In vEcoli, restoring TU00434 / TU00435 by strengthening the dedup key caused fitPromoterBoundProbability to fail with Solver could not find optimal value — the parca's gene-level expression targets become over-constrained when split across three co-TUs of the same gene. The vEcoli report suggests fixing build_vector_k / build_matrix_G in fit_sim_data_1.py to distribute targets across co-TUs. This has not been attempted in v2ecoli yet; the same failure mode is expected because the parca fitting code is a direct port.

8. Next steps

  1. Run multigeneration_report.py on the plasmid cache to observe the DnaG trajectory across generations in v2ecoli and produce a figure analogous to Fig. 10 of the vEcoli report. This is the first place we can quantify whether the v2ecoli depletion rate matches vEcoli's (6 → 0–1 over 5–6 generations).
  2. Port the tf_tu_exemption partial fix. Simplest route: implement it as an inline sim_data transform inside save_cache (in v2ecoli/composite.py near line 174), no variants registry needed. Default exemption: {"PC00010": ["TU00352"]} (v2ecoli only has TU00352 to exempt because of Cause 4).
  3. Fix the growth_limits realize-Array bug so multigen can run in-place (without fresh-composite rebuild). Shortest fix likely: annotate listeners/growth_limits as listener_store (a subclass of InPlaceDict) so the schema resolver doesn't try to pack its 39 heterogeneous sub-fields into a single np.array. Cross-reference the growth_limits schema in v2ecoli/processes/polypeptide_elongation.py:254–480.
  4. Investigate the build_vector_k / build_matrix_G co-TU distribution. This is the only fix that actually restores TU00434 and TU00435 (rather than merely papering over the symptom with tf_tu_exemption on the surviving TU). Likely blocked behind the variants/fitting pipeline port.
  5. Epistemic note. Fixes 2 and 3 change different layers: #2 modifies sim_data post-parca (surgical, reversible); #4 modifies the parca fitting (principled). Both deserve to be gated behind a feature flag so we can A/B the effect on DnaG depletion rate and on the broader 773-gene population that Cause 3 is estimated to affect.
