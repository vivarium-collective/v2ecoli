Phase 3 multi-parameter extensibility

This sprint demonstrates the Phase-3 pipeline (emission → aggregate → persist → load → distance metric → noise-floor calibration → ABC acceptance) generalizes to multi-dimensional inference without changing the infrastructure. Each new parameter is one config knob exposed via the composite generator + one threading commit; the rest of the pipeline (zarr persistence, mean-to-mean ε calibration, sequential refinement) carries through unchanged. Scaling to >10 parameters is straightforward; the dominant cost remains forward-sim wall time, not pipeline overhead.
