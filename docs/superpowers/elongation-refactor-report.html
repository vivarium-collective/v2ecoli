Polypeptide elongation: strategy classes → wireable processes

The three elongation models — Base, TranslationSupply, SteadyState — were a hidden strategy-class inheritance chain selected by config flags. They are now three sibling PartitionedProcess classes you choose by wiring, each declaring only the ports its math touches. Baseline behaviour is preserved bit-for-bit.

The three processes

Base BasePolypeptideElongationmax-rate

Elongates polypeptides at the media-determined maximum ribosome rate, limited only by the amino acids the allocator granted.

Builds the upcoming codon sequences for every active ribosome and runs polymerize over the allocated amino-acid pool to maximise total elongation; terminates completed proteins. No supply gating, no tRNA charging, no ppGpp.

Key formulas
  • sequences = buildSequences(proteinSequences, positions, v·dt)
  • result = polymerize(sequences, aa_allocated, …) → Δlength, aa_used
  • GTP cost = gtpPerElongation · nElongations (gtpPerElongation = base + 2: ATP→AMP charging hydrolyses folded into GAM since charging is not modelled)
ports: 10 in · 25 out Defines the five hooks (elongation_rate, amino_acid_counts ≡ identity, request, final_amino_acids ≡ identity, evolve).
Wire it: PARTITIONED_PROCESSES["ecoli-polypeptide-elongation"] = BasePolypeptideElongation

TranslationSupply TranslationSupplyPolypeptideElongationsupply-gated

Base elongation, but the amino acids available each tick are capped by a media-driven supply rate scaled to cell mass.

Overrides only elongation_rate (media table) and amino_acid_counts (caps requested AAs by the supply target). Inherits Base's request/final_amino_acids/evolve. Same port surface as Base.

Key formulas
  • aa_supply = supply_rate(media) · elngRateFactor · dry_mass · dt · N_A
  • aa_counts = min(aa_supply, aasInSequences) (element-wise cap)
ports: 10 in · 25 out Overrides elongation_rate + amino_acid_counts; inherits the rest from Base.
Wire it: PARTITIONED_PROCESSES["ecoli-polypeptide-elongation"] = TranslationSupplyPolypeptideElongation

SteadyState SteadyStatePolypeptideElongationcharging + ppGpp · baseline default

Full mechanistic model: explicit aminoacyl-tRNA charging and ppGpp stringent-response regulation couple translation speed to tRNA availability and growth state. This is what the baseline wires.

Solves the tRNA-charging steady state and ppGpp kinetics each tick; the minimum charged fraction across amino acids throttles the effective elongation rate. Adds the boundary input and the full growth_limits charging/supply listener surface (27 extra output leaves).

Key formulas
  • f_charged_a = [charged_tRNA_a] / ([charged_tRNA_a] + [uncharged_tRNA_a])
  • v_eff ∝ min_a f_charged_a (translation speed coupled to tRNA availability)
  • d[ppGpp]/dt = k_RelA·[RelA]·[unchg]/(KD+[unchg]) − k_SpoT·[SpoT]·[ppGpp]/(KI+[ppGpp]) + k_SpoT_syn·[SpoT]
  • GTP cost = gtpPerElongation · nElongations (NO +2 — charging modelled explicitly)
ports: 11 in · 52 out Overrides elongation_rate / request / final_amino_acids / evolve and adds _amino_acid_supply, _ppgpp_request, _ppgpp_evolve, distribution_from_aa; overrides inputs()/outputs() to union the charging/ppGpp ports.
Wire it: PARTITIONED_PROCESSES["ecoli-polypeptide-elongation"] = SteadyStatePolypeptideElongation

Port divergence — the payoff

Base and TranslationSupply share the same surface (10 inputs · 25 outputs) — Supply only changes the math. SteadyState declares 11 inputs · 52 outputs: the extra boundary input plus the 27 charging / ppGpp / AA-supply growth_limits listeners below. The wiring now shows exactly what each model touches instead of one process declaring the union.
